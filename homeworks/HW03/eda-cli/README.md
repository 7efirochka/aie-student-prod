# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

## Команды

- `overview` - команда быстрого просмотра таблиц и минимальной сводки по ней (включая саму таблицу);
- `report` - полный и детальный анализ таблицы включающий проверку метрик и составление диаграмм;
- `sample` - выводит 10 (по умолчанию) случайных записей из датафрейма

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
uv run eda-cli report data/example.csv --max-hist-columns 5
uv run eda-cli report data/example.csv --title "Мой умный отчёт" --top-k-categories 4
uv run eda-cli report data/example_2.csv --min-missing-share 0.3

```

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.

## Добавленные эвристики

- `has_constant_columns` - показывает есть ли столбцы в которых все значения одинаковые (от таких колонок мало толку, можно не учитывать);
- `has_high_cardinality_categoricals` - показывает есть ли столбцы с большим количеством различных значений (снижает эффективность датафрейма, большая вероятность, что модель переобучиться);
- `has_suspicious_id_duplicates` - показывает есть ли в столбце user_id дубликаты (столбец должен быть только с уникальными значениями, если это не верно, модель может неверно обучиться).


## Добавленные параметры CLI-команды

Для `report`:

- `--max-hist-columns` - количество числовых колонок в гистограмме;
- `--top-k-categories` - количество top - значений для категоральных признаков;
- `--title` - заголовок отчёта;
- `--min-missing-share` - порог определения проблемных столбцов в зависимости от количества пропусков;
- `--json-summary` - создаст json файл с небольшой информацией о датасете.

Для `sample`:

- `--n` - количество выводимых значений
